# Topic Cluster Model

A pillar page targets a broad keyword. Spoke pages target narrow, long-tail variants. Every spoke links UP to the pillar. The pillar links DOWN to each spoke.

## Rules for clustering

1. **Pillar = the broadest, highest-volume keyword** of the group.
2. **Spokes = narrower variants** that share search intent with the pillar.
3. **A keyword belongs to a cluster if its top-10 SERP overlaps the pillar's top-10 by 30%+.** Without SERP data, fall back to lexical similarity + intent match.
4. **3-8 spokes per pillar.** Fewer than 3 isn't a cluster, just a page. More than 8 dilutes the pillar.
5. **No spoke should outrank its pillar.** If a spoke targets a higher-volume keyword, it should BE the pillar.

## Internal links (enforced by Internal Linking Assistant)

- Every spoke MUST link to its pillar from the body using descriptive anchor text.
- Pillar SHOULD link to every spoke from a "Related" or "Read more" section.
- Spokes MAY cross-link to each other when contextually relevant.
