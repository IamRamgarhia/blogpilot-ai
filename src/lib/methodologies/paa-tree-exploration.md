# PAA Tree Exploration & Content Mapping

Recursively expanding People-Also-Ask boxes reveals the full question space around a topic. Use the tree to design clusters, find quick-win posts, and prove topical authority.

## Process

1. **Seed** with the broadest viable head term ("wordpress seo", not "yoast seo settings").
2. **Expand 3 levels deep** with throttled scraping (3-5s between requests).
3. **Cap at ~40 questions** total. Beyond that returns diminishing-value variants.
4. **Cluster** the leaves by topic: lexical overlap ≥ 30% on key nouns + same SERP intent.
5. **Tag each leaf with intent:** informational / commercial / transactional.
6. **Prioritize by:**
   - Quick wins (questions with ≤ 5 indexed answers — low competition)
   - Pillar candidates (questions appearing at multiple levels — central concept)
   - Differentiators (questions only 1-2 competitors answer)

## Output decisions per node

For each terminal question, decide one of:
- **Standalone post** (high search volume, distinct intent) → add to calendar
- **FAQ section item** on a parent pillar post (low volume, related intent) → add to FAQ
- **Skip** (duplicate of an existing tree question or weak commercial value)

## Anti-patterns

- Don't write a post for every leaf. The tree often has 30+ nodes; a healthy cluster is 1 pillar + 5-12 spokes.
- Don't treat all PAA as equal. Some are auto-generated by Google and have no real search behind them.
- Don't expand a question if it's already at level 3 — diminishing returns + may get rate-limited.
